# 競馬買い目下書きアプリ PRD

> ※ アプリ名は未定（仮称：競馬買い目下書きアプリ）

---

## 1. 概要

### 1.1 コンセプト

**競馬仲間に買い目を送るまでの、予想の下書き帳。**
印・メモ・買い目を1レース分まとめて整理し、テキストで共有できる。

### 1.2 解決したい課題

- 競馬の買い目をLINEで友達に共有する時、毎回手打ちで整形するのが面倒
- 三連単フォーメーションなど、点数と合計金額の暗算ミスが起きやすい
- 予想を立てる過程（印、気づいたこと）が紙やメモアプリに散らばって残らない
- 実際の購入はJRA公式／IPATで行うため、購入前の「下書き」を整理する場が欲しい

### 1.3 ターゲットユーザー

- 自分自身（一次ユーザー）
- 予想や買い目をLINEでやり取りする競馬仲間（共有相手）

## 2. スコープ

### 2.1 MVP対象

- レース作成・管理
- 予想（印、馬名、メモ）
- 買い目作成（馬連・馬単・三連複・三連単／通常・BOX・フォーメーション・流し）
- 点数と合計金額の自動計算
- テキスト整形＆共有（コピー／LINE）
- 履歴（レース一覧がそのまま履歴）

### 2.2 対象外（MVPでは作らない）

- 的中・不的中の記録、回収率集計（netkeibaのマイレース等に委ねる）
- 単勝・複勝・枠連・ワイド・WIN5
- 画像による共有
- SNS機能・他ユーザーとの繋がり
- netkeiba／JRAデータの自動連携
- 予想の自動生成（印からの買い目自動生成など）
- 馬名の予測変換機能

## 3. ユーザー操作フロー

```
① アプリを開く
   ↓
② ホーム画面（レース一覧 兼 履歴）
   ├─ 新規レース追加（競馬場・R・レース名・出走頭数をインライン入力）
   ├─ 既存レースをタップ → レース詳細へ
   └─ 既存レース長押し → 複製 / 再共有 / 削除
   ↓
③ レース詳細画面（デフォルトは予想タブ）
   ├─ レース全体メモ（常駐・常時編集可）
   ├─ 予想タブ
   │   └─ 馬番ごとに印、馬名（任意）、メモ（任意）を入力
   └─ 買い目タブ
       └─ 買い目を追加／編集／削除
   ↓
④ 買い目追加フロー
   券種選択 → 買い方選択 → 馬番入力（直接タップ or 印から呼び出し）
   → 1点あたり金額入力（デフォルト100円）
   → 点数×金額の自動計算 → 警告判定 → 一覧に追加
   ↓
⑤ 共有
   買い目一覧から共有ボタン → テキスト生成 → コピー／LINE共有
   ↓
⑥ 完了（データはローカルに保存され、ホーム画面に履歴として残る）
```

## 4. 画面設計

### 4.1 ホーム画面（レース一覧 兼 履歴）

レース一覧と履歴は統合された1画面。

**表示内容：**
- 画面上部：アプリタイトル、新規レース追加ボタン（＋）
- レース一覧：新しい順に並ぶ
  - 各レース行：日付、競馬場、R、レース名、買い目の要約（例：「三連複F 他2件」）
- タップ：レース詳細へ遷移
- 長押し：メニュー表示（複製 / 再共有 / 削除）

**新規レース追加：**
- ＋ボタンを押すとインラインで入力欄が展開
- 入力項目：競馬場、R、レース名、出走頭数
- 画面遷移せずその場で作成

### 4.2 レース詳細画面

**構造：**

```
┌────────────────────────────────┐
│ ← 東京11R 日本ダービー           │ ヘッダー
├────────────────────────────────┤
│ 📝 レース全体メモ                │ 常駐・常時編集可
│ （自由記述テキストエリア）         │
├────────────────────────────────┤
│ [予想] [買い目]                  │ タブ
├────────────────────────────────┤
│                                │
│ （タブの中身）                    │
│                                │
└────────────────────────────────┘
```

**レース全体メモ：**
- レース詳細画面の最上部に常駐（タブ切替でも表示され続ける）
- 自由記述テキストエリア
- 内容が長くなると縦方向に自動で伸びる（折りたたみなし）
- 共有テキストには含まれない（自分用メモ）

### 4.3 予想タブ

馬番ごとに印・馬名・メモを管理。

**表示内容：**
- 出走頭数分の行が並ぶ（例：1〜18）
- 各行に以下の要素：
  - 馬番
  - 印（◎ ○ ▲ △ ☆ ✕ の中から1つ選択、未選択OK）
  - 馬名（任意入力）
  - メモ（任意入力、タップで入力欄が開く）
- メモは長文OK、その馬の行の高さが可変で伸びる

**レイアウト方針：**
- 1頭1ブロック、縦並び
- メモが長くなっても崩れない構造
- 印未設定・馬名未入力・メモ未入力の馬はシンプルに表示

### 4.4 買い目タブ

そのレースの買い目一覧と追加フォーム。

**表示内容：**
- 既に作成した買い目がカード形式で一覧表示
  - 各カード：券種、買い方、馬番、点数、1点金額、合計金額
  - 各カードに編集・削除ボタン
- 画面下部に「買い目を追加」ボタン
- 画面最下部に買い目合計金額を表示

**買い目追加フロー：**
1. 「買い目を追加」ボタン
2. 券種選択（馬連／馬単／三連複／三連単）
3. 買い方選択（券種により選択肢が変わる、詳細は第6章）
4. 馬番入力
   - 直接タップ：馬番ボタンをタップして選ぶ
   - 印から呼び出し：「◎を軸に入れる」「○▲△を相手に入れる」等のショートカット
5. 1点あたり金額入力（デフォルト100円）
6. 自動計算結果を確認（点数・合計金額）
7. 警告判定を経て一覧に追加

### 4.5 共有シート

**表示内容：**
- 共有テキストのプレビュー
- 「コピー」ボタン
- 「LINEで共有」ボタン

共有テキストフォーマットは第7章参照。

## 5. 機能仕様

### 5.1 レース管理

| 項目 | 内容 |
|------|------|
| 作成 | 競馬場、R、レース名、出走頭数を入力 |
| 編集 | レース情報の後からの編集を可能にする |
| 削除 | レース自体を削除 |
| 複製 | 既存レースの内容（印、メモ、買い目）を複製 |
| 再共有 | 過去レースから再び共有テキストを生成 |

### 5.2 予想

| 項目 | 内容 |
|------|------|
| 印 | ◎（本命）、○（対抗）、▲（単穴）、△（連下）、☆（連下下）、✕（消し） |
| 印の割り当て | 1頭につき印は1つ、同じ印を複数頭に付けるのは可 |
| 馬名 | 任意入力、未入力でも問題なし |
| 馬番ごとメモ | 任意入力、長文OK |
| レース全体メモ | 任意入力、長文OK、レース詳細画面最上部に常駐 |

### 5.3 買い目

| 項目 | 内容 |
|------|------|
| 券種 | 馬連、馬単、三連複、三連単 |
| 買い方 | 通常、BOX、フォーメーション、流し |
| 馬番入力 | 直接タップ／印から呼び出しの2方式 |
| 1点あたり金額 | デフォルト100円、変更可（100円単位） |
| 1レース内の買い目数 | 複数追加可 |
| 編集・削除 | 作成済みの買い目それぞれに対して可 |

### 5.4 共有

| 項目 | 内容 |
|------|------|
| 共有単位 | 1レースごと |
| 形式 | テキスト（標準モード：馬番＋馬名＋印＋買い目、メモ非含有） |
| 共有方法 | クリップボードコピー、LINE共有 |

### 5.5 履歴

| 項目 | 内容 |
|------|------|
| 保存先 | ローカルストレージ |
| 一覧 | ホーム画面にレース一覧として統合表示（新しい順） |
| 操作 | 詳細表示、複製、再共有、削除 |

## 6. 点数・金額計算ロジック

### 6.1 基本ルール

- **合計金額 = 点数 × 1点あたり金額**
- 1点あたり金額は100円単位、デフォルト100円
- 計算は入力が完了した時点で即時反映

### 6.2 券種別・買い方別の点数計算

#### 馬連（2頭・順不同）

| 買い方 | 入力 | 点数 |
|--------|------|------|
| 通常 | 2頭指定 | 1点 |
| BOX | n頭選択 | nC2 = n(n−1)/2 |
| フォーメーション | 1頭目候補 A、2頭目候補 B | A×B の中で順不同重複を除いた組み合わせ数 |
| 流し | 軸1頭、相手n頭 | n点 |

#### 馬単（2頭・着順あり）

| 買い方 | 入力 | 点数 |
|--------|------|------|
| 通常 | 1着・2着を指定 | 1点 |
| BOX | n頭選択 | nP2 = n(n−1) |
| フォーメーション | 1着候補 A、2着候補 B | A×B のうち同じ馬が重複するケースを除外 |
| 1着流し | 軸1頭（1着固定）、相手n頭 | n点 |
| 2着流し | 軸1頭（2着固定）、相手n頭 | n点 |

#### 三連複（3頭・順不同）

| 買い方 | 入力 | 点数 |
|--------|------|------|
| 通常 | 3頭指定 | 1点 |
| BOX | n頭選択 | nC3 = n(n−1)(n−2)/6 |
| フォーメーション | 3つの枠 A、B、C | A×B×C のうち3頭全て異なるケースを順不同重複除いてカウント |
| 軸1頭流し | 軸1頭、相手n頭 | nC2 = n(n−1)/2 |
| 軸2頭流し | 軸2頭、相手n頭 | n点 |

#### 三連単（3頭・着順あり）

| 買い方 | 入力 | 点数 |
|--------|------|------|
| 通常 | 1着・2着・3着を指定 | 1点 |
| BOX | n頭選択 | nP3 = n(n−1)(n−2) |
| フォーメーション | 1着候補 A、2着候補 B、3着候補 C | A×B×C のうち3頭全て異なるケースをカウント |
| 1着流し | 軸1頭（1着固定）、相手n頭 | nP2 = n(n−1) |
| 2着流し | 軸1頭（2着固定）、相手n頭 | nP2 = n(n−1) |
| 3着流し | 軸1頭（3着固定）、相手n頭 | nP2 = n(n−1) |
| 1・2着流し | 軸2頭（1-2着）、相手n頭（3着候補） | 2n点（軸の順序2通り × 相手n頭） |
| 2・3着流し | 軸2頭（2-3着）、相手n頭（1着候補） | 2n点 |

### 6.3 警告表示

**トリガー条件（いずれか）：**
- 点数が 100点 を超えた
- 合計金額が 10,000円 を超えた

**表示内容：**
- 「点数が多くなっています。内容を確認してください」のようなダイアログ
- 「OK（そのまま追加）」「キャンセル」の2択

誤操作による高額買い目の事故を防ぐのが目的。

## 7. 共有テキストフォーマット

### 7.1 フォーマット（標準モード）

```
📍 [競馬場][R] [レース名]

◎[馬番] [馬名]
○[馬番] [馬名]
▲[馬番] [馬名]
△[馬番,馬番,馬番]

[券種略記] [買い方] [馬番表現] / [点数]点×[金額]円 = [合計]円
[券種略記] [買い方] [馬番表現] / [点数]点×[金額]円 = [合計]円

合計: [合計金額]円
```

### 7.2 表記ルール

**券種略記：**
- 馬連 → 馬連
- 馬単 → 馬単
- 三連複 → 3連複
- 三連単 → 3連単

**買い方略記：**
- 通常 → （何も付けない）
- BOX → BOX
- フォーメーション → F
- 流し → 流し（着付け指定がある場合は「1着流し」等）

**馬番表現：**
- 単独：`7`
- 複数：`1,3,5`
- 軸と相手：`7-1,3,5,9,12`
- フォーメーション：`1,3-5,7-9,12,14` のように着ごとにハイフン区切り

**馬名は印が付いている馬のみ表示。**
印なしの馬番は馬番のみで表示（馬名省略）。
馬名が未入力の場合は馬番のみ表示。

### 7.3 出力例

```
📍 東京11R 日本ダービー

◎7 イクイノックス
○3 ドウデュース
▲5 タイトルホルダー
△1,9,12

3連複F 7-3,5-1,9,12 / 6点×200円 = 1,200円
馬連BOX 3,5,7 / 3点×500円 = 1,500円

合計: 2,700円
```

## 8. データ永続化

- 保存先：ローカルストレージ（端末内）
- 保存タイミング：各入力の都度、自動保存
- バックアップ・同期：MVPでは非対応（ローカルのみ）

## 9. 非機能要件

- モバイルファースト（スマホでの操作性を最優先）
- PWA対応（ホーム画面に追加してアプリ的に使える）
- オフライン動作（ローカル完結のため、ネット接続不要）
